# CHANGELOG - Historial de cambios\n\n## [1.0.0] - 2026-06-22\n\n### ✨ Características nuevas\n\n#### Endpoint de inicialización de admin\n- **POST `/api/v1/admin/init`**: Permite crear el primer usuario admin sin autenticación\n- Solo funciona si no existe ningún admin previo\n- Valida email único y contraseña mínimo 8 caracteres\n- Status: 201 (Created), 409 (Conflict si ya existe admin)\n\n#### Autenticación y autorización\n- JWT con HS256 y expiración de 30 minutos\n- Roles: admin, veterinario, cliente\n- Control de acceso basado en rol (RBAC) en todos los endpoints\n- Bcrypt hashing para contraseñas\n\n#### Endpoints de administración\n- `GET /api/v1/admin/dashboard`: Panel del admin\n- `GET /api/v1/admin/summary`: Resumen con 13 métricas operacionales\n- `POST /api/v1/admin/veterinarios`: Crear veterinarios\n- `POST /api/v1/admin/clientes`: Crear clientes\n- `PATCH /api/v1/admin/usuarios/{user_id}/estado`: Activar/desactivar usuarios\n\n#### Endpoints de clientes\n- `GET /api/v1/clientes/dashboard`: Panel del cliente\n- `GET /api/v1/clientes/me`: Ver su perfil\n- `GET /api/v1/clientes/{cliente_id}`: Ver detalles de cliente\n- `GET /api/v1/clientes/mis-establecimientos`: Ver sus establecimientos\n\n#### Endpoints de veterinarios\n- `GET /api/v1/veterinarios/dashboard`: Panel del veterinario\n- `GET /api/v1/veterinarios/me`: Ver su perfil\n- `GET /api/v1/veterinarios/{veterinario_id}`: Ver detalles de veterinario\n- `GET /api/v1/veterinarios/clientes`: Ver sus clientes\n- `GET /api/v1/veterinarios/{veterinario_id}/clientes/{cliente_id}/establecimientos`: Ver establecimientos de un cliente\n\n#### Endpoints de establecimientos y bebederos\n- `GET /api/v1/establecimientos/{establecimiento_id}`: Detalles con bebederos\n- `GET /api/v1/establecimientos/{establecimiento_id}/bebederos`: Lista de bebederos\n- `GET /api/v1/bebederos/{bebedero_id}`: Detalles con monitoreos e imágenes\n\n#### Endpoints de autenticación\n- `POST /api/v1/auth/register`: Registrar como cliente\n- `POST /api/v1/auth/login`: Login y obtener JWT\n- `GET /api/v1/auth/me`: Ver usuario autenticado\n\n### 🏗️ Arquitectura\n\n- **Layered architecture**: core → db → models → schemas → crud → services → routers\n- **Type hints**: Python 3.12+ con forward references\n- **Eager loading**: selectinload para evitar N+1 queries\n- **Transactional safety**: flush + commit pattern en operaciones críticas\n- **Dependency injection**: FastAPI dependencies para auth y permisos\n\n### 📊 Base de datos\n\n**Modelos implementados**:\n1. `Usuario`: email único, password_hash, rol, activo, timestamps\n2. `Veterinario`: usuario FK, especialidad, telefono, ubicacion, foto_perfil\n3. `Cliente`: usuario FK, veterinario FK, razon_social, contacto\n4. `Establecimiento`: cliente FK, nombre, ubicacion, timestamps\n5. `Bebedero`: establecimiento FK, nombre, ubicacion, ip, puerto, estado\n6. `MonitoreoDiario`: bebedero FK, nivel_agua, distancia, cobertura, estado sensores\n7. `Imagen`: monitoreo FK, nombre_archivo, ruta, tamano, checksum\n8. `Evento`: bebedero FK, tipo, descripcion, gravedad, resuelta\n\n### 🔐 Seguridad\n\n- **Hashing**: bcrypt con salt automático\n- **Tokens**: JWT HS256 con exp, sub, email, role, nombre\n- **Validación**: Pydantic en todas las entradas\n- **RBAC**: Control de acceso por rol en router + service layer\n- **Email único**: Constraint en BD + validación aplicación\n- **Bootstrap seguro**: /admin/init solo para primer admin\n\n### 📝 Documentación\n\nArchivos agregados:\n- `README.md`: Guía completa de arranque y endpoints\n- `AUTHENTICATION.md`: Flujo de auth, JWT, estructura de token\n- `RBAC.md`:
